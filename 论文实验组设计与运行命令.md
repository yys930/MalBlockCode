# MalBlock 论文实验组设计与运行命令

本文档基于当前项目代码与论文最新定位整理。

**论文核心定位：**

> 本系统不是让大模型替代 Suricata 做恶意流量检测，而是在 Suricata 已给出告警证据后，由大模型负责阻断决策、执行方式选择与响应控制。

因此，论文实验应围绕以下问题展开：

1. 大模型是否能做出有效的阻断决策；
2. RAG 是否能提升阻断决策效果；
3. 系统是否能完成检测—决策—执行闭环；
4. 执行阻断后，后续风险是否得到压制。

---

## 1. 建议保留的实验组

| 实验组 | 名称 | 通道 | 主要目的 | 是否建议写入论文主结果 |
|---|---|---|---|---|
| G1 | 无 RAG 阻断决策基线实验 | CSV | 评估无历史案例辅助时的决策有效性 | 是 |
| G2 | 有 RAG 阻断决策增强实验 | CSV | 验证 RAG 是否提升决策有效性 | 是 |
| G3 | Offline 阻断决策闭环实验 | Offline PCAP | 验证真实检测输入下的决策与执行闭环 | 是 |
| G4 | Replay 在线联动实验 | Replay | 验证在线回放条件下的执行联动能力 | 是 |
| G5 | 执行 / 无执行 对照实验 | Replay | 验证真实执行阻断后的收益压制效果 | 是 |
| S1 | Top-k 工程补充实验 | Offline / Replay | 说明 token 成本控制与工程优化取舍 | 可选附录 |

## 2. 每组实验重点评估什么

## G1：无 RAG 阻断决策基线实验

### 实验目的
验证在 **没有历史案例检索** 的情况下，大模型仅依据当前告警证据，能否做出有效阻断决策。

### 推荐设置
- 通道：`csv`
- 数据：`mixed_eval_cleaned.csv`
- `topk=0`
- `selection_mode=priority`
- `rag_top_k=0`

### 对应评估维度
- 决策有效性评估
- 数据覆盖性评估（可作为支撑项）

### 建议关注指标
- `decision_eval.risk_detection_metrics.precision`
- `decision_eval.risk_detection_metrics.recall`
- `decision_eval.risk_detection_metrics.f1`
- `decision_eval.strong_mitigation_metrics.precision`
- `decision_eval.strong_mitigation_metrics.recall`
- `decision_eval.strong_mitigation_metrics.f1`
- `decision_eval.csv_metrics.matched_decisions`
- `dataset_summary.selected_rows`

---

## G2：有 RAG 阻断决策增强实验

### 实验目的
验证引入 **历史案例检索（RAG）** 后，大模型的阻断决策效果是否提升。

### 推荐设置
- 与 G1 保持一致
- 唯一变化：`rag_top_k=3`

### 对应评估维度
- 决策有效性评估
- 数据覆盖性评估（可作为支撑项）

### 与 G1 的核心对比指标
- `risk_detection_metrics.precision / recall / f1`
- `strong_mitigation_metrics.precision / recall / f1`
- `matched_decisions`

### 论文解释重点
这一组的重点是说明：

> **RAG 是否帮助模型做出更准确、更稳定的阻断决策。**

---

## G3：Offline 阻断决策闭环实验

### 实验目的
验证在 **真实 PCAP 经 Suricata 检测后**，系统是否能够基于检测证据完成阻断决策，并在执行层形成完整闭环。

### 推荐设置
- 通道：`offline`
- `topk=0`
- `rag_top_k=3`
- 建议至少跑 2~3 个代表性 PCAP：
  - `Tuesday-WorkingHours.pcap`
  - `Wednesday-workingHours.pcap`
  - `Thursday-WorkingHours.pcap`

### 对应评估维度
- 执行有效性评估
- 干预收益观察评估
- 数据覆盖性评估（可选）

### 建议关注指标
- `dataset_summary.input_rows`
- `dataset_summary.selected_rows`
- `decision_eval.decision_count`
- `decision_eval.action_distribution`
- `execution_eval.tool_success_count`
- `execution_eval.new_enforcement_count`
- `execution_eval.repeat_enforcement_count`
- `execution_eval.decision_to_execution_consistency`
- `effect_eval.mean_suppression_ratio`

### 建议辅助查看文件
- `channel_summary.json`
- `summary.json`
- `evaluation_report.json`

---

## G4：Replay 在线联动实验

### 实验目的
验证在 **在线回放场景** 下，系统能否稳定完成“回放—检测—决策—执行”的联动流程。

### 推荐设置
- 通道：`replay`
- 回放速率：`10x`
- `topk=20`
- `rag_top_k=3`
- 建议使用便于演示的 PCAP：
  - `Thursday-WorkingHours.pcap`

### 对应评估维度
- 执行有效性评估
- 干预收益观察评估（代理观察）
- 数据覆盖性评估（可选）

### 建议关注指标
- `stream_alert_count_raw`
- `stream_alert_count_filtered`
- `stream_batches`
- `selected_groups`
- `decision_count`
- `execution_eval.tool_success_count`
- `execution_eval.repeat_enforcement_ratio`
- `execution_eval.skipped_execution_count`
- `execution_eval.decision_to_execution_consistency`

### 建议辅助查看文件
- `summary.json`
- `alerts_filtered.jsonl`
- `tcpreplay.stdout.log`
- `tcpreplay.stderr.log`
- `suricata.log`

---

## G5：执行 / 无执行 对照实验

### 实验目的
在相同回放流量、相同参数条件下，对比“真实执行阻断”和“DRY_RUN 不执行”两组结果，验证真实执行阻断后，后续风险是否被压制。

### 对应评估维度
- 干预收益观察评估
- 执行有效性评估（辅助）

### 重点对比文件
- 执行组：`llm_decisions.jsonl`
- 执行组：`alerts_filtered.jsonl`
- 对照组：`alerts_filtered.jsonl`
- 两组：`summary.json`
- 两组：`channel_summary.json`

### 建议重点比较的数据
1. 同一 `src_ip` 在决策后 TTL 范围内的后续告警数；
2. 执行组 vs 对照组的后续 `after_hits` 差异；
3. 平均压制增益；
4. 完全压制率；
5. 匹配决策数。

## 3. 正式实验前的统一准备

建议在项目根目录运行：

```bash
cd /home/os/FinalCode/malblock
source backend/MBvenv/bin/activate
export SILICONFLOW_API_KEY=你的密钥
```

### DRY_RUN 设置
- 调试阶段建议：

```bash
export DRY_RUN=1
```

- 正式执行实验（G3、G4、G5 执行组）建议：

```bash
unset DRY_RUN
```

- 对照实验中的“无执行组”需要：

```bash
export DRY_RUN=1
```

## 6. 推荐实验顺序

建议按以下顺序推进：

1. **先跑 G1、G2**：先证明 RAG 是否提升决策有效性；
2. **再跑 G3**：得到真实检测输入下的执行闭环结果；
3. **再跑 G4**：证明系统在线联动能力；
4. **最后跑 G5**：补最强的干预收益证据。

---

## 7. 最终建议

如果时间有限，论文主结果至少应包含：

- G1：无 RAG 决策有效性基线实验
- G2：有 RAG 决策有效性增强实验
- G3：Offline 执行闭环实验
- G4：Replay 在线联动实验

如果你想把“阻断是否真的产生收益”写得更扎实，则必须补上：

- G5：执行 / 无执行 对照实验
